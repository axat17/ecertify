You are picking up an in-progress engineering project called ReleaseIQ — an ADP internal release management platform. All code has been written and is being handed to you. Read this entire prompt before doing anything.


== PROJECT OVERVIEW ==

ReleaseIQ is a full-stack platform for managing Salesforce and React release certifications, Jira sync, Copado bundle tracking, and team coordination. It consists of two separate repositories:

1. releaseiq-microservice — Node.js + Express + TypeScript REST API
2. releaseiq-ui — React 18 + Vite + TypeScript frontend

There is also a single-file HTML prototype (ReleaseIQ_v2.html) that represents the full UI vision with dummy data and localStorage simulation — useful as a reference for any UI work.


== TECH STACK ==

Microservice:
- Node.js + Express + TypeScript
- MongoDB + Mongoose (local install, no Docker)
- JWT auth + Azure AD MSAL (mock mode for local dev)
- Winston logging, node-cron, express-rate-limit
- Environments: LOCAL -> DIT -> FIT -> IAT -> PROD

UI:
- React 18 + Vite + TypeScript
- TanStack React Query v5 (server state)
- Zustand (client/auth state, persisted to localStorage)
- Axios with JWT interceptor + auto-retry on 502/503/504
- React Router v6 with lazy loading

== KEY DESIGN DECISIONS — DO NOT CHANGE WITHOUT REASON ==

1. Cert session UUID = share link credential.
   /cert/:sessionId has NO auth guard by design. BU testers who may not have Azure AD
   must be able to land on the join screen. The join form collects their identity.
   The resume endpoint GET /api/certifications/:sessionId/resume is also unauthenticated.

2. Sessions persist in MongoDB, not in browser state.
   Closing the browser, navigating away, losing the tab — none of these end a session.
   The durationSeconds field is recalculated from startedAt on every resume call.
   The UI useTimer hook re-derives elapsed time from startedAt on mount.

3. Lost cert link recovery flow.
   ActiveSessionsPage polls GET /api/certifications/active every 30s.
   The Link button copies window.location.origin + /cert/ + sessionId to clipboard.
   Release Managers paste this into Slack to resend to testers.
   Force Complete closes stuck sessions when a tester's window is unrecoverable.

4. Join is idempotent — duplicate sessions are prevented.
   POST /api/certifications/join checks for an existing In Progress session with the same
   certifierEmail + projectId + releaseVersion before creating a new one.
   If found, it returns the existing session with resumed: true.

5. Jira sync is pull-on-demand, served from MongoDB cache.
   All story reads come from MongoDB (JiraStory collection).
   No Jira API calls happen during normal read operations.
   Stale data is served during sync failures — the UI shows a yellow banner.
   project.syncStatus drives the banner: never / syncing / synced / error / stale

6. Two required Jira labels per story for Copado.
   Label 1: CopadoCICD — links story to Copado pipeline
   Label 2: release label (e.g. Salesforce_26.14) — scopes story to this release
   All 8 teams share the EXACT same release label name.
   DevOps manually creates Copado bundles from stories that have both labels.

7. Consolidated + drill-down team view.
   JiraFixVersion stores per-team aggregates (done/inProgress/todo counts).
   GET /api/jira/fix-versions/:projectId returns consolidated totals + perTeam array.
   UI shows consolidated by default; each team card expands to individual stories.

8. Error handling pattern for all external API calls.
   Retry 3 times with exponential backoff: 1s, 2s, 3s.
   On final failure: set project.syncStatus = error, log to ErrorLog collection, throw.
   Admin Error Log page shows all failures with resolve button.

9. Secret masking.
   API tokens are stored in MongoDB but returned to the client as "••••••••".
   The frontend only sends a new token value if the user types something new.
   The save handler checks: if value === '••••••••', keep existing DB value.

10. common/ is the import boundary in the UI.
    Feature components in apps/ import from @/common/components, @/common/hooks,
    @/common/utils, @/common/types only.
    They never import from sibling feature folders.
    This keeps features decoupled and common/ as the single source of truth for UI primitives.

11. Global CSS keyframes are injected once in App.tsx as a <style> tag.
    riq-pulse  — used on LIVE dot badges and active session indicators
    riq-spin   — used on Button loading spinner
    riq-fadeup — used on toasts and modal entrance
    All inline animation references use these names. Do not rename them.

12. React Query cache keys follow a strict pattern.
    ['projects']                     — all projects list
    ['project', projectId]           — single project
    ['sessions', projectId, status]  — sessions filtered by project + status
    ['activeSessions', projectId]    — active sessions for ActiveSessionsPage poll
    ['certSession', sessionId]       — single cert session by UUID
    ['stories', projectId]           — Jira stories for a project
    ['fixVersions', projectId]       — fix version aggregates
    ['bundles', projectId]           — Copado bundles
    ['adminRequests']                — admin access requests
    ['errorLogs']                    — error log entries
    ['allUsers']                     — all users (main admin only)
    ['activity']                     — platform activity feed
    Invalidate the correct key after any mutation. Never use a different key for the same data.

== RULES FOR THIS CHAT ==

1. Before modifying any feature, read the relevant section of FEATURES.md first.

2. Always import shared UI from @/common/ — never write inline Button or Badge components
   inside apps/. If a new reusable component is needed, add it to common/ first.

3. Never add auth to GET /api/certifications/:sessionId/resume or the /cert/:sessionId route.
   This must remain public. It is intentional by design (see design decision 1).

4. Always follow the existing error handling pattern: withRetry -> ErrorLog -> stale data fallback.
   Do not add try/catch that swallows errors silently.

5. Use the established React Query cache key convention (listed above). If adding a new query,
   follow the pattern ['entity', ...params].

6. When adding a new UI component that will be reused across features, put it in common/.
   When adding a component that belongs to exactly one feature, put it in that feature's
   components/ subfolder.

7. The microservice config is read from .env.{NODE_ENV} automatically. Never hardcode
   environment-specific values. Always use config.X from src/config/index.ts.

8. When connecting real Jira or Copado APIs, change only the mock=false branch in the routes.
   The mock branch should remain intact as a fallback for local development.
